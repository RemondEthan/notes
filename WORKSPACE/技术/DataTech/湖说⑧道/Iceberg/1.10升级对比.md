## **Apache Iceberg 1.9 系列核心总结**

| 类别           | 内容                                                         |
| -------------- | ------------------------------------------------------------ |
| **发布时间线** | - **1.9.0**: 2025-04-28（含新功能 + bug 修复） - **1.9.1**: 2025-05-27（纯 bug 修复） - **1.9.2**: 2025-07-16（关键 bug 修复，防表损坏） |

### 1.9.0引入新功能

#### 1. **数据类型增强**

- 新增`variant` 逻辑类型（类似 JSON/动态类型），支持：
  - Avro / Parquet / ORC 的读写
  - 提取表达式（`extract`）、toString、默认值
  - 指标（metrics）收集
- 新增 地理空间类型：`geometry` 和 `geography`
- 支持 纳秒级时间戳（timestamp(9)）
- 支持 未知类型（unknown） 的安全处理

#### 2. **规范（Spec）更新**

- 允许 Equality Deletes 与 Row Lineage 共存（V3 表）
- 明确 Row Lineage 是 V3 表升级的强制要求
- 更新快照 ID 行为：V3+ 表中 `current-snapshot-id` 可为 null
- 支持在 V3 中使用 `source-id`

#### 3. **性能与运维**

- 新增 分区统计信息（Partition Stats）读写器（#11216）
- 引入 FileRewritePlanner 接口，分离重写规划与执行逻辑
- 新增 MetricsReporter 用于快照管理监控
- 支持 批量删除过期快照（`RemoveSnapshots`）

#### 4. **Catalog & 安全**

- 新增 Auth Manager API（为未来细粒度权限控制铺路）
- 支持 view-override catalog 属性
- REST Catalog 默认移除 `/HEAD` 端点（更安全）

#### 5. **生态系统集成**

- Spark
  - 将 V2 删除文件（Delete Vectors）自动重写为 V3 格式
  - 新增 `_row_id` 和 `_last_updated_sequence_number` 虚拟列读取
- Flink
  - 支持 `CREATE TABLE LIKE`
  - 支持 watermark 用于窗口 SQL
  - 支持 Avro/Parquet 的 timestamp(9)、variant、默认值
- Kafka Connect
  - 新增 SMT（单消息转换器）支持 Debezium 和 AWS DMS
- AWS
  - 集成 S3 Analytics Accelerator 库（提升元数据性能）

## **Apache Iceberg 1.10.0 核心概览**

| 类别             | 关键内容                                                     |
| ---------------- | ------------------------------------------------------------ |
| **发布时间**     | 2025-09-11                                                   |
| **定位**         | 功能增强 + 生产稳定性提升 + 多引擎深度集成                   |
| **推荐升级场景** | 使用 Spark 4.0 / Flink 2.0、Row-level Delete、REST Catalog、加密、地理数据、Parquet 高级特性 |

### 1.10.0引入新功能

### **一、重大新功能与增强**

#### 1. **安全与加密**

- 新增表级加密支持
  - 表元数据中可存储 加密密钥信息（Spec & API & REST）
  - AWS/GCP 提供 KeyManagementClient 实现
- REST 客户端增强
  - 支持 TLS 配置、HTTP 代理、自定义 User-Agent
  - 支持 禁用 Token 刷新、共享 Auth 刷新执行器

#### 2. **Row-Level Delete 与 V3 表完善**

- V3 表全面启用 Row Lineage（不再可选）
- 清理孤儿 Delete Vectors (DVs)
  - 重写数据文件时自动传播并删除悬空 DV
  - 跟踪待删除文件以清理孤儿 DV
- Spark/Flink 全面支持 V3 行级更新语义

#### 3. **性能与运维工具**

- 分区统计（Partition Stats）增强
  - 支持 增量刷新
  - 支持 DV 和 timestamp(9)
  - 新增 存储过程/Action 计算分区统计
- 文件重写控制
  - 可配置 最大重写文件数（`max-number-of-files-rewrite`）
  - 支持 自定义分区排序（`RewriteManifest`）
- 资源清理优化
  - `ExpireSnapshots` 支持 显式清理过期元数据（`cleanExpiredMetadata`）
  - `BaseTransaction` 使用 批量删除 清理未提交文件

#### 4. **数据类型与格式支持**

- Parquet 升级至 1.16.0
  - 支持 `variant` 逻辑类型注解
  - 支持 DELTA_BINARY_PACKED 编码
  - 支持 字典编码的 UUID
- Arrow/Parquet 读取器重构
  - 支持 纳秒时间戳（timestamp(9)）
  - 修复 混合编码页的直接内存泄漏
- Spark 4.0 支持 `unknown` 类型读写

### **二、引擎集成亮点**

####  **Spark**

- 正式支持 Spark 4.0
  - 迁移存储过程到 Spark 内建实现
  - 利用 Spark 4.0 的 条件置空机制 实现行血缘
- 性能优化
  - 存储分区连接（Storage Partitioned Join）支持 bucket/hour→day 归约
  - 流式微批处理中 `maxRecordsPerMicroBatch` 改为 软限制
- Bug 修复
  - 修复含标识字段（identifier fields）的 DML 失败问题
  - 修复 `ADD COLUMN WITH DEFAULT` 抛出明确异常（暂不支持）

####  **Flink**

- 支持 Flink 2.0（移除 Flink 1.18）
- 全新动态 Sink（v2）
  - 支持 动态 Schema & 分区演进
  - 自动创建/写入多表（fan-out）
  - 默认并行度继承输入流（利于 chain）
  - 支持 Compaction 和 Rewrite Data Files
- 表维护增强
  - 支持 ZooKeeper 锁
  - 支持 删除孤儿文件

#### **云厂商集成（AWS / Azure / GCP）**

- AWS
  - 支持 多存储凭证前缀
  - 修复 S3 连接/内存泄漏
  - 添加 LegacyMd5Plugin（兼容旧 S3 网关）
- Azure
  - 支持 access token 认证（`adls.token`）
  - 修复 凭证刷新并发问题
- GCP
  - 新增 BigQuery Metastore Catalog
  - 支持 Google 身份认证 和 多凭证前缀

### **三、重要行为变更与弃用**

| 类型         | 变更说明                                                     |
| ------------ | ------------------------------------------------------------ |
| **行为变更** | Hive Catalog：查询不存在的 namespace 时 **抛出 `NoSuchNamespaceException`**（原返回空列表） |
| **弃用移除** | - 移除 Flink 1.18 支持 - 移除 `DynConstructors/DynMethods` 中自 1.7 起废弃的 public 方法 - 清理各模块中标记为 1.10 移除的废弃代码 |
| **规范澄清** | - 明确写入时防止孤儿 DV 的要求 - 澄清地理对象在 min/max bounds 中的行为 |

### **四、关键 Bug 修复**

- REST Catalog
  - 停止对 502/504 重试 → 防止表元数据损坏（延续 1.9.2 修复）
  - 修复初始化失败时资源未关闭
- Core
  - 修复 JDBC Catalog 初始化竞态条件
  - 修复分区演进 + null 分区值导致的元数据不完整
  - 修复 `ExpireSnapshots` 增量清理逻辑错误
- Parquet/Arrow
  - 修复直接内存泄漏（混合编码页）
  - 修复嵌套列的指标列限制问题
